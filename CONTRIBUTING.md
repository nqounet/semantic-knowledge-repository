# Contributing to Semantic Knowledge Repository

First off, thank you for considering contributing to Semantic Knowledge Repository! It's people like you that make Semantic Knowledge Repository such a great tool.

## Code of Conduct

This project and everyone participating in it is governed by the [Semantic Knowledge Repository Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How Can I Contribute?

### Reporting Bugs

This section guides you through submitting a bug report for Semantic Knowledge Repository.

- **Check if the bug is already reported** in the [Issues](https://github.com/nqounet/semantic-knowledge-repository/issues) section.
- **Provide clear steps to reproduce** the issue, including what you expected to happen and what actually happened.
- Include details about your environment (OS, Node.js version, Gemini CLI version).

### Suggesting Enhancements

If you have an idea to improve Semantic Knowledge Repository, we'd love to hear it!

- Open an issue describing the feature.
- Explain **why** this feature would be useful.
- Provide examples of how it might be used.

### Pull Requests

1. **Fork the repo** and create your branch from `main`.
2. **Make your changes**. If you've added code that should be tested, add tests.
3. **Ensure the code lints** (if applicable).
4. **Create a Pull Request** with a clear description of the changes.

## Development Setup

1. Clone your fork of the repository:
   ```bash
   git clone https://github.com/YOUR_USERNAME/semantic-knowledge-repository.git
   ```
2. Navigate to the project directory:
   ```bash
   cd semantic-knowledge-repository
   ```
3. Install dependencies (if any are added in the future):
   ```bash
   npm install
   ```
4. Test your changes locally before submitting a Pull Request.
